🎯 **What:** The `generate_simple_tab` function within `TablatureGenerator` lacked test coverage. Additionally, while the task rationale indicated the function handled empty arrays natively with an early return (`if not midi_notes: return []`), the actual implementation in `src/chorderizer/generators.py` was missing this safeguard. This PR adds the missing early return to the source code and implements a comprehensive suite of unit tests for the function.

📊 **Coverage:**
The newly added tests in `tests/test_generators.py` cover:
- **Happy Path:** Validates correct formatting, header text, and accurate guitar string-to-fret assignments based on provided MIDI notes (e.g., C chord).
- **Empty Notes:** Verifies the function handles an empty array gracefully, successfully returning an empty array per the early return logic.
- **Out of Bounds:** Confirms notes that fall outside the maximum allowable frets (15) are ignored and left empty in the tablature output.

✨ **Result:** Enhanced test coverage and reliability for the tablature generation feature. Added a necessary guard clause to prevent potential edge case errors.
